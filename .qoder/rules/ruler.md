    1. 只读先行
任何对 PATH 的修改前，必须先读取当前系统 PATH 并回显给用户确认。
    2. 禁止覆盖
    禁止使用 setx PATH 'xxx;%PATH%' 这类“整串替换”写法；只能使用追加单项的方式：
    ○ 图形界面：通过“编辑系统环境变量”→“新建”条目
    ○ PowerShell：powershell复制
    $old=[Environment]::GetEnvironmentVariable('PATH','Machine')
    $new=$old+';E:\jdk17\bin'
    [Environment]::SetEnvironmentVariable('PATH',$new,'Machine')
    ○ 必须确保追加前长度 < 1024 字符，否则提示用户手动处理。
    3. 事前提示
    任何 PATH 修改命令执行前，必须明确告知用户：
    ○ 将要添加的路径
    ○ 当前 PATH 长度/字符数
    ○ 是否可能截断风险
    4. 事后核验
    修改完成后立即回显新的系统 PATH，让用户肉眼检查是否完整。
    5. 一键回滚
    每次修改前自动生成 .reg 备份文件（或至少复制原 PATH 到剪贴板），并告知用户回滚命令。
    6. 默认拒绝：若用户未明确同意“修改 PATH”，则默认拒绝执行任何 PATH 相关写操作。
