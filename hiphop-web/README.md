# 说唱聚合 | HipHop Hub
&gt; 中文说唱的一站式“微博＋演出日历＋音乐人主页”

---

## 1. 项目背景
- 中文说唱圈信息极度分散：  
  – B 站/抖音只有零散视频；  
  – 公众号文章各自为营，搜“说唱”关键词，一天能刷出 30 个号；  
  – 各地 Livehouse 演出信息藏在“××HipHop”私人号推文里，错过是常态。
- 新人音乐人没有“公共广场”可发歌，只能私信公众号求转发，效果随机。
- 粉丝想系统浏览“今天说唱圈发生了什么”，只能多平台来回跳转。

一句话：**中文说唱需要自己的“首页”**。

---

## 2. 我的想法（MVP → 社区）
### Phase 1  聚合
- 爬虫＋RSS 订阅 30+ 主流公众号、B 站说唱 UP、微博超话，自动抓取：  
  – 文章 / 视频 / MV 首发  
  – 演出信息（关键字：live、巡演、站次、购票链接）
- 算法去重＋人工标签（地域、厂牌、风格），生成“今日说唱速递”时间线。

### Phase 2  发布
- 任何注册用户可上传：  
  – 单曲 / Beat / Lyrics 视频  
  – 演出海报（自动同步到“演出日历”）
- 作品自动出现在“新人榜”与对应风格标签页，解决“零粉发歌没人听”问题。

### Phase 3  社区
- 点赞 / 评论 / 收藏 / 转发（支持生成卡片分享到微信）
- “圈子”功能：地域圈（川渝、粤港澳…）、风格圈（BoomBap、Trap、Drill…）
- 演出日历支持“我想去”报名，主办方可后台导出名单，打通票务。

### Phase 4  移动端
- PWA 套壳 → 推送“你关注的音乐人发新歌了”
- 小程序版本（演出日历+扫码验票）

---

## 3. 长期愿景
让中文说唱生态里“信息流动”与“新人曝光”这两件事，不再靠运气和人情，而靠一套公开、透明、可检索的聚合协议。  
未来开放 API，任何播放器、播客、NFT 平台都能接入“中文说唱元数据”。
